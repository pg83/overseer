## Final ritual before declaring READY

Trunk advances while you work — other tickets get merged into TRUNK_PATH's master. Before declaring READY, you must:

1. **Squash** all your commits on this branch since base into ONE commit:

       git reset --soft "$(git merge-base HEAD origin/HEAD)"
       git commit -m "<single descriptive message for this ticket>"

   Use the ticket DESCR as the basis for the commit message; expand with what was done. After this, `git log origin/HEAD..HEAD --oneline` should show exactly ONE commit.

2. **Rebase** onto trunk's current master (TRUNK_PATH from the input above):

       git fetch "$TRUNK_PATH" master
       git rebase FETCH_HEAD

   Resolve conflicts if any (`git add <files>; git rebase --continue`). Re-run tests after rebase to confirm the merge of trunk's new state with your change still works.

3. Final verification:

       git log FETCH_HEAD..HEAD --oneline   # MUST be exactly one commit, yours
       git status                           # MUST be clean

Re-run the rebase until both checks pass. Only then finish with the READY verdict.

## Finish with one verdict event

Finish with exactly one `verdict` event — a JSON line, not prose. Don't close with a prose summary; put it in the verdict's `detail`. The verdict is the last line you emit.

    {"type": "verdict", "verdict": "READY", "detail": "<one-line summary of what you did>"}
    {"type": "verdict", "verdict": "CANT_DO", "detail": "<reason>"}
    {"type": "verdict", "verdict": "ALGEDONIC", "detail": "<what is fucked and why nothing local fixes it>"}

- READY — work done, committed, squashed, rebased onto current master, tests pass.
- CANT_DO — *this ticket as defined* cannot be completed; it goes to the arbiter for a local decision (rework / re-implement / escalate).
- ALGEDONIC — the **emergency cord**. Pull it only when the problem is *bigger than this ticket*: the plan or goal is wrong, a prerequisite is fundamentally missing, the ticket keeps you in a loop nothing at your level can break, or you've honestly hit a wall and need a rethink from the top. It bypasses review/merge/arbiter and wakes the replanner in its algedonic context for a full root-cause analysis. Not a louder CANT_DO — reserve it for systemic trouble, and say concretely in `detail` what's wrong.
